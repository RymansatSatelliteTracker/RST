# pr-create スキル

現在のブランチの変更内容を分析し、GitHub向けのプルリクエストを作成します。

## 手順

以下のステップを順番に実行してください。

### 1. 現状確認（並列実行）

次のコマンドを並列で実行して現状を把握する：

- `git status` — 未コミットの変更を確認
- `git log develop..HEAD --oneline` — mainブランチからの全コミット一覧を確認
- `git diff develop...HEAD` — mainブランチからの全差分を確認
- `git branch --show-current` — 現在のブランチ名を確認

### 2. 変更内容の分析

収集した情報をもとに以下を分析する：

- **変更の種類**: 新機能追加 / バグ修正 / リファクタリング / テスト / ドキュメント / その他
- **変更の範囲**: 影響するコンポーネント・モジュール
- **変更の目的**: なぜこの変更が必要か（コミットメッセージやブランチ名から推測）
- **関連Issue番号**: ブランチ名やコミットメッセージから `#数字` を抽出

### 3. PRタイトルと説明文の作成

分析結果をもとに日本語でPRタイトルと説明文を作成する：

**タイトル規則:**
- 70文字以内
- 変更の種類と内容を簡潔に表現
- Issue番号があれば末尾に `(#番号)` を付与

**説明文の構成:**
```
## 概要
<!-- 変更の目的と背景を2〜3文で説明 -->

## 変更内容
<!-- 箇条書きで主要な変更点を列挙 -->

## テスト方法
<!-- 動作確認の手順または確認済みのテスト内容 -->

## 関連Issue
<!-- 関連するIssue番号（例: closes #123） -->
```

### 4. リモートへのプッシュ確認

ローカルブランチがリモートに存在するか確認する：

```bash
git remote -v
git branch -r | grep $(git branch --show-current)
```

未プッシュの場合は `git push -u origin HEAD` でプッシュする（ユーザーに確認してから実行）。

### 5. PRの作成

`gh pr create` コマンドでPRを作成する：

```bash
gh pr create --base develop --title "PRタイトル" --body "$(cat <<'EOF'
## 概要
...

## 変更内容
...

## テスト方法
...

## 関連Issue
...

EOF
)"
```

### 6. 完了報告

作成したPRのURLをユーザーに提示する。

## 注意事項

- **ベースブランチ**: 通常は `develop` ブランチをターゲットにする（プロジェクトの設定に従う）
- **未コミットの変更がある場合**: PRを作成する前にユーザーに確認する
- **破壊的変更がある場合**: 説明文に `⚠️ 破壊的変更` セクションを追加する
- **PRは作成前に必ず内容をユーザーに確認する**
