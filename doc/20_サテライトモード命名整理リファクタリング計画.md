# サテライトモード命名整理リファクタリング計画

## 背景

RST では「サテライトモード」という用語が、現在コード上で複数の意味に使われています。

1. 無線機の運用モード
   - `SATELLITE`
   - `SPLIT`
   - `UNSET`
2. 衛星運用時の Normal / Reverse 設定
   - `NORMAL`
   - `REVERSE`

一方で、既存の永続化データでは以下のキー名がすでに使用されています。

- `frequency.json`
- `default_satellite.json`
- `app_config.json`

これらのファイルでは `satelliteMode` というキーが **Normal / Reverse を表す意味** で利用されています。
このキー名は配布済みデータおよびユーザーローカル保存データとの互換性の観点から、変更コストが高い状態です。

そのため本リファクタリングでは、**既存データ形式を維持しつつ、内部コード上の意味衝突を減らす** ことを目的とします。

---

## 目的

- 既存JSONのキー名（`satelliteMode`, `enableSatelliteMode`）を変更しない
- アプリ内で `satelliteMode` の意味を **Normal / Reverse** に揃える
- `SATELLITE / SPLIT / UNSET` は別名に分離し、概念衝突を避ける
- UI文言と設定データの意味を揃える
- 既存ユーザーデータとの後方互換性を維持する

---

## 命名方針

### 1. 永続化データのキー名は維持する

以下は変更しない。

- `enableSatelliteMode`
- `satelliteMode`

このうち `satelliteMode` は、今後も **Normal / Reverse** を表すキーとして扱う。

### 2. 無線機の `SATELLITE / SPLIT / UNSET` は別名に分離する

現在 `Constant.Transceiver.SatelliteMode` が持っている次の値:

- `SATELLITE`
- `SPLIT`
- `UNSET`

は、今後 `SatelliteOperationMode` として扱う。

### 3. `NORMAL / REVERSE` を `SatelliteMode` として扱う

UIおよび永続化データでの用語に合わせ、次の値:

- `NORMAL`
- `REVERSE`

は `SatelliteMode` として扱う。

### 4. 変数名も意味に合わせて整理する

例:

- `satelliteMode`（SATELLITE/SPLIT を指しているもの）
  - → `satelliteOperationMode`
- `isSatelliteMode`
  - → `isSatelliteOperationEnabled`
- `satTrackMode`
  - → `satelliteMode`
- `isSatTrackingModeNormal`
  - → `isSatelliteModeNormal`

---

## 変更方針の整理

### 変更しないもの

- `frequency.json` のキー名
- `default_satellite.json` のキー名
- `app_config.json` のキー名
- 既存ユーザーデータの保存形式

### 変更するもの

- `Constant.ts` の定数名
- 内部の状態保持変数名
- コメントおよび型コメント
- `SATELLITE / SPLIT` を表すUI内部変数名

---

## 主な変更対象

### 1. 定数定義

対象:

- `src/common/Constant.ts`

変更内容:

- `Constant.Transceiver.SatelliteMode`
  - `SATELLITE / SPLIT / UNSET`
  - → `Constant.Transceiver.SatelliteOperationMode`
- `Constant.Transceiver.TrackingMode`
  - `NORMAL / REVERSE`
  - → `Constant.Transceiver.SatelliteMode`

### 2. 永続化モデル・型

対象:

- `src/common/model/AppConfigModel.ts`
- `src/common/types/satelliteSettingTypes.ts`
- `src/renderer/components/molecules/SatelliteInfoEditorDialog/SatelliteInfoEditorDialogForm.ts`

変更内容:

- プロパティ名 `satelliteMode` は維持する
- コメントを「Normal / Reverse を表す」意味に修正する

### 3. 設定編集UI

対象:

- `src/renderer/components/molecules/SatelliteInfoEditorDialog/*`

変更内容:

- `form.satelliteMode` は維持する
- 参照先を `Constant.Transceiver.SatelliteMode.NORMAL / REVERSE` に変更する

### 4. アクティブ衛星設定の保持

対象:

- `src/renderer/service/ActiveSatServiceHub.ts`

変更内容:

- `satTrackMode` を廃止し、`satelliteMode` に寄せる
- `satelliteMode: boolean` のような曖昧な名前を `isSatelliteOperationEnabled` に変更する

### 5. 無線機制御UI・ロジック

対象:

- `src/renderer/components/organisms/TransceiverCtrl/useTransceiverCtrl.ts`
- `src/renderer/components/organisms/TransceiverCtrl/TransceiverCtrl.vue`
- `src/renderer/components/organisms/TransceiverCtrl/useSatelliteModeStateManager.ts`

変更内容:

- `satelliteMode`（SATELLITE/SPLIT を保持している変数）
  - → `satelliteOperationMode`
- `isSatelliteMode`
  - → `isSatelliteOperationEnabled`
- `isSatTrackingModeNormal`
  - → `isSatelliteModeNormal`
- 定数参照先を新命名に合わせて更新する

## 影響範囲

### 永続化互換性

- 既存JSONのキー名は変更しないため、保存済みデータとの互換性は維持できる

### 実装影響

- 定数参照名の変更により、レンダラー側の参照箇所に広く影響する
- 状態保持名の変更により、Composable / Service / Vue テンプレート間の参照更新が必要になる

### テスト影響

特に以下を重点確認する。

- 衛星設定画面のバリデーション
- アクティブ衛星設定の反映
- TransceiverCtrl の Satellite / Split 切替
- Normal / Reverse の切替挙動
